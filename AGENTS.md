# AGENTS.md

## 1. 目标定义（Objective）

本项目用于辅助完成毕业论文写作，Agent必须严格遵守学术规范，确保：

- 内容真实、可验证，不得编造数据或文献
- 结构符合标准毕业论文格式
- 表达规范、逻辑清晰、术语准确
- 所有输出可直接用于论文或稍作修改即可使用
- Agent 的核心职责：生成可以通过毕业答辩的论文内容

***

## 2. 全局约束（Global Constraints）

### 2.1 严禁行为

Agent **禁止**：

-  编造参考文献（必须真实存在）
-  编造实验数据、测试结果
-  使用“可能、大概、应该”等模糊表述（学术场景）
-  输出口语化表达（如：比如说、其实、然后等）
-  抄袭已有论文原文（必须重写表达）
-  输出与当前章节无关内容
-  擅自扩展未被要求的系统功能或设计

***

### 2.2 必须遵守

Agent **必须**：

-  使用正式学术语言（偏论文风格）
-  结构层级清晰（严格对应目录）
-  所有技术描述符合实际开发情况（SSM / Vue / MySQL 等）
-  与用户提供的系统保持一致（不得虚构架构）
-  使用第三人称叙述（避免“我”、“我们”）
-  生成论文相关的文件“（包括读取文档的脚本）都要放在“论文”文件夹下；正文放在论文\内容下
- 论文框架来源于核心\论文框架.txt，项目核心技术来源于核心\项目计划书-一种基于云端协同和姿态识别算法的运动动作矫正系统.pdf
- 输出的文档必须是md格式，不允许使用emoji表情
- 在正文中引用的参考文献必须来源于内容\第七章\_参考文献.md

***

## 3. 输出格式规范（Output Format）

### 3.1 段落要求

- 每段 150～300 字
- 避免单句成段
- 段落之间逻辑递进清晰

***

### 3.2 列表规范

仅在以下场景允许使用列表：

- 功能点说明
- 系统模块划分
- 测试用例

否则优先使用段落表达。

***

## 4. 技术一致性约束（Tech Consistency）

默认技术栈：

- 前端（C端）：微信小程序 (Native)
- 前端（B端）：Vue 3 + Element Plus
- 后端：Python + Flask
- 视觉算法：MediaPipe + OpenCV
- AI 大模型：DeepSeek API
- 数据存储：MySQL

***

## 6. 参考文献规则（References）

- 仅在用户要求时生成
- 文献必须来自知网数据库[中国知网](https://www.cnki.net/)
- 在文档里生成后必须在聊天框内补充文献的链接
- 格式符合 GB/T 7714
