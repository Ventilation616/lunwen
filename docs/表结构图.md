# 数据库表结构详细图

## 1. 表结构单独图

### 1.1 sys_user (用户表)

```mermaid
erDiagram
    SYS_USER {
        BIGINT id PK "用户ID"
        VARCHAR username "用户名"
        VARCHAR password_hash "密码哈希值"
        VARCHAR openid "微信开放标识"
        VARCHAR unionid "微信统一标识"
        VARCHAR avatar_url "头像URL"
        VARCHAR phone "手机号"
        TINYINT gender "性别(0未知,1男,2女)"
        INT age "年龄"
        FLOAT height "身高(厘米)"
        FLOAT weight "体重(千克)"
        VARCHAR nick_name "昵称"
        VARCHAR role "角色(super,admin,user)"
        VARCHAR status "状态(active,inactive)"
        DATETIME created_at "创建时间"
        DATETIME updated_at "更新时间"
        TINYINT is_deleted "逻辑删除标记"
    }
```

### 1.3 sport_record (运动记录表)

```mermaid
erDiagram
    SPORT_RECORD {
        BIGINT id PK "记录ID"
        BIGINT user_id FK "用户ID"
        BIGINT sport_type_id FK "运动类型ID"
        INT duration "运动时长(秒)"
        INT correct_count "正确次数"
        INT incorrect_count "错误次数"
        JSON detail_json "详细数据(骨架点坐标,关节角度等)"
        DATETIME created_at "创建时间"
        DATETIME updated_at "更新时间"
        TINYINT is_deleted "逻辑删除标记"
    }
```
